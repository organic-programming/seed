# TASK08 — Document Distributed Mesh and Transport Security

## Objective

Update the Organic Programming specification documents to cover the distributed mesh topology (`op mesh`), mTLS transport security, per-listener security policy, and the mesh registry (`mesh.yaml`).

## Reference Documents

- [DESIGN_mesh.md](../v0.8/DESIGN_mesh.md) — `op mesh` commands, data model, SDK integration
- [DESIGN_transport_rest_sse.md](../v0.6/DESIGN_transport_rest_sse.md) — REST + SSE transport analysis
- [DESIGN_public_holons.md](../v0.9/DESIGN_public_holons.md) — per-listener security policy and public holon exposure

## Changes

### 1. `OP.md` — Add `op mesh` Command Reference

**Location**: new section 11 after "Serve & Dial", renumber subsequent sections.

- 11.1 Overview (operator tool, pure Go, small-scale design boundary)
- 11.2 `op mesh init` (CA generation)
- 11.3 `op mesh add <host> [--deploy]` (cert generation + SSH provisioning)
- 11.4 `op mesh remove <host>` (cert deletion)
- 11.5 `op mesh list` (topology view)
- 11.6 `op mesh status` (health check via mTLS gRPC)
- 11.7 `op mesh describe <host>` (holon enumeration)
- 11.8 `mesh.yaml` format (schema reference)

---

### 2. `PROTOCOL.md` — Add Transport Security Section

**Location**: new section after current §7 "Summary".

- 8.1 Threat model (eavesdropping, MITM, impersonation)
- 8.2 mTLS for gRPC (mutual certificates, TLS 1.3)
- 8.3 Certificate identity (CN, SAN, peer extraction)
- 8.4 Local transports — no TLS (`stdio://`, `unix://`, `mem://`)
- 8.5 Per-listener security modes (`none` / `mesh` / `public`)
- 8.6 SDK integration (`serve.Run` auto-detection, `connect` mTLS)
- 8.7 Future: application-level E2EE

---

### 3. `PROTOCOL.md` — Add Mesh Discovery to Section 2

- Remote Discovery via Mesh Registry (`discover` reads `mesh.yaml`, queries `HolonMeta/Describe`)

### 4. `CONVENTIONS.md` — Add `~/.op/mesh/` to Standard Directories

Add `~/.op/mesh/` and its contents to the standard directories table.

### 5. `MESH_YAML.md` — New Specification Document

**Location**: `organic-programming/MESH_YAML.md` (new file, alongside `HOLON_YAML.md`)

- Scope: *"Who are the other hosts in the network?"*
- Full schema: `ca`, `hosts[]` (address, port, cert, added)
- Remote host file layout: `~/.op/mesh/{ca.crt, host.key, host.crt}`
- Auto-generated by `op mesh` — not hand-edited
- Relationship to SDK (`discover` + `connect`)
- Include three-YAML cross-reference table

### 6. `HOLON_YAML.md` — Add Per-Listener Security

- Add `serve.listeners` schema with `uri`, `security`, `auth` fields
- Add `serve.tls` schema (cert/key paths for public listeners)
