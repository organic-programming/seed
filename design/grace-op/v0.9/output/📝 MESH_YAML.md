# mesh.yaml ‚Äî Mesh Registry

The mesh registry declares the **network topology** ‚Äî which hosts
participate in the mesh, how to reach them, and where their
certificates are stored.

`mesh.yaml` is auto-generated by `op mesh` commands. It is not
hand-edited.

---

## Scope

*"Who are the other hosts in the network?"*

| Question | Answered by |
|---|---|
| What hosts are in the mesh? | `hosts[]` |
| How to reach each host? | `hosts[].address` + `hosts[].port` |
| How to verify each host? | `hosts[].cert` (signed by mesh CA) |
| Who is the trust anchor? | `ca.cert` |

---

## Location

```
~/.op/mesh/mesh.yaml       # on the operator's machine
```

The operator's machine holds the full registry (all hosts). Remote
hosts only need their own credentials (`ca.crt`, `host.key`,
`host.crt`) ‚Äî they do not have `mesh.yaml`.

---

## Schema

```yaml
ca:
  cert: ~/.op/mesh/ca.crt
  created: 2026-03-09T12:00:00Z

hosts:
  - address: paris.example.com
    port: 9090
    cert: ~/.op/mesh/hosts/paris.example.com/host.crt
    added: 2026-03-09T12:01:00Z

  - address: lyon.example.com
    port: 9090
    cert: ~/.op/mesh/hosts/lyon.example.com/host.crt
    added: 2026-03-09T12:02:00Z

  - address: bordeaux.example.com
    port: 9443
    cert: ~/.op/mesh/hosts/bordeaux.example.com/host.crt
    added: 2026-03-09T12:03:00Z
```

### `ca` (required)

| Field | Type | Required | Description |
|---|---|---|---|
| `ca.cert` | string | yes | Path to CA public certificate |
| `ca.created` | datetime | yes | ISO 8601 timestamp of CA creation |

### `hosts[]` (required)

| Field | Type | Required | Description |
|---|---|---|---|
| `address` | string | yes | Hostname or IP address |
| `port` | integer | yes | gRPC listen port (default: 9090) |
| `cert` | string | yes | Path to host's certificate (signed by CA) |
| `added` | datetime | yes | ISO 8601 timestamp when host was added |

## Lifecycle

| Event | Command | Effect on `mesh.yaml` |
|---|---|---|
| Create mesh | `op mesh init` | File created, empty `hosts` |
| Add host | `op mesh add <host>` | Entry appended to `hosts` |
| Remove host | `op mesh remove <host>` | Entry removed |
| Renew cert | `op mesh add <host> --force` | Entry updated (new cert path) |

---

## SDK Usage

The SDK reads `mesh.yaml` during `discover` to find remote holons:

1. Parse `~/.op/mesh/mesh.yaml`
2. For each host, query `HolonMeta/Describe` over mTLS
3. Cache results (TTL: configurable, default: 60s)
4. Return matching holon with remote address

The SDK never writes to `mesh.yaml`. Only `op mesh` commands
modify it.

## Rules

- `mesh.yaml` is **auto-generated** by `op mesh` ‚Äî do not hand-edit.
- `ca.key` is the root of trust ‚Äî losing it means re-initializing
  the entire mesh.
- `ca.key` must **never** be deployed to remote hosts.
- At the target scale (~10 hosts), `mesh.yaml` stays under 50 lines
  and is fully human-readable.
