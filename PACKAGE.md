## 1. Overview
The `.holon` package directory is the universal runtime and distribution unit for Organic Programming.

**Core Principle:** The source `.proto` file carrying `(holons.v1.manifest)` is the single source of truth. All `op` operations consume the `.holon` packages derived from it.

> **Related Specs:** [OP.md](OP.md) (CLI), [HOLON_BUILD.md](HOLON_BUILD.md) (Build), [PROTO.md](PROTO.md) (Manifest), [DISCOVERY.md](../../DISCOVERY.md) (Discovery resolution algorithm).

## 2. Package Structure
A package contains any combination of source, built artifact, or compiled binary. It must have at least one.

```text
<slug>.holon/
  .holon.json              # Generated identity cache (always present if processed)
  bin/<arch>/<slug>        # Pre-built native executable (e.g. darwin_arm64)
  dist/                    # Bundled/transpiled artifact + runtime deps
  git/                     # Cached remote source tree fetched as a dependency (only in $OPPATH/cache)
```

### Artifact Layers
| Layer | Characteristics | Examples |
|-------|-----------------|----------|
| `bin/` | Native executable, platform-specific (`<os>_<cpu>`). No runtime needed. | `go build`, `cargo` |
| `dist/` | Code + deps. Platform-independent. Interpreter required. | `tsc`, `pip`, `npm` |
| `git/` | Remote source tree fetched by `op mod`. Build required before execution. | `op mod pull` |

## 3. Metadata Cache (`.holon.json`)
`.holon.json` accelerates discovery. **Generated by `op build` or `op discover`, never manually edited.**

```json
{
  "schema": "holon-package/v1",
  "slug": "gabriel-greeting-go",
  "uuid": "3f08b5c3-8931-46d0-847a-a64d8b9ba57e",
  "identity": { "given_name": "Gabriel", "family_name": "Greeting-Go", "motto": "..." },
  "lang": "go",
  "runner": "go-module",
  "status": "draft",
  "kind": "native",
  "transport": "stdio",
  "entrypoint": "gabriel-greeting-go",
  "architectures": ["darwin_arm64", "linux_amd64"],
  "has_dist": false,
  "has_source": true
}
```

* **`runner`**: Determines execution (`go-module`, `cargo`, `python`, `node`, etc.).
* **`transport`**: Default transport hint (`stdio`, `tcp`).
* **`entrypoint`**: Executable name in `bin/` or script in `dist/`.

## 4. Lifecycle & Workflows
Packages maintain identical structure across all environments.

| Environment | Location | Typical Content |
|-------------|----------|-----------------|
| **Source** | Dev repository | Raw source files (no `.holon` wrapper) |
| **Build** | `.op/build/<slug>.holon/` | `.holon.json`, `bin/` or `dist/` |
| **Cache** | `$OPPATH/cache/<pkg>/` | `.holon.json`, `git/`, maybe `bin/`/`dist/` |
| **Install** | `$OPBIN/<slug>.holon/` | `.holon.json`, target-arch `bin/` or `dist/` |
| **Bundle** | `App.app/.../Holons/` | `.holon.json`, target-arch `bin/` or `dist/` |

### Command Behaviors
* **`op build`**: Reads `.proto` -> compiles tree -> outputs `bin/` or `dist/` -> generated `.holon.json`.
* **`op install`**: Copies package to `$OPBIN/` (compiles `git/` first if no applicable binary/dist exists).
* **`op uninstall`**: Removes `<slug>.holon/` from `$OPBIN/`.
* **`op run <slug>`**: Reads `runner` from `.holon.json` and launches:
  * Compiled: `<pkg>/bin/<arch>/<entrypoint> serve --listen <uri>`
  * Interpreted: `<interpreter> <pkg>/dist/<entrypoint> serve --listen <uri>`
* **`op man`**: Extracts docs directly from `.proto` or running binary. Exports `.md` via `--export-path`.
* **`op inspect`**: Introspects static contract via `protocompile`[^1] (source) or live contract via `Describe` RPC (binary).

## 5. Dependency Management (`op mod`)
Uses Minimum Version Selection (MVS) to populate `$OPPATH/cache`.

* **`holon.mod`**: Declares dependencies with minimum versions.
* **`holon.sum`**: Integrity hashes for fetched packages.
* **Commands**: `init`, `add`, `remove`, `tidy`, `pull`, `update`, `list`, `graph`.
* **Fetch (`pull`)**: Checks cache -> downloads (registry or git) -> verifies `holon.sum` -> builds on-demand when run if needed.

## 6. Composite Bundles (`.app`)
Holon packages can be fully embedded inside platform deliverables like macOS `.app`.

1. **Path:** `Contents/Resources/Holons/<slug>.holon/`
2. **Content:** Embed only the target architecture's binaries.
3. **Execution:** Host SDK discovers via `.holon.json`, launches processes, and manages lifecycle (start, probes, cleanup).
4. **Network:** Inside bundles, transports are strictly local (`stdio://` or `tcp://localhost:<port>`).
5. **Signing:** Embedded `bin/` executables must be included in the host app code signature.

**Recipe Steps (`holon.mod` bundle process):**
* `copy_artifact`: Embeds a newly built package.
* `use_installed`: References an installed package from `$OPBIN`.
* `use_cached`: Embeds a dependency fetched to `$OPPATH/cache`.

## 7. SDK Contract
To be fully self-describing, SDKs must expose package metadata at runtime.

1. **Embedding:** `op build` generates a `DescribeResponse` snapshot. Compiled holons embed this at link time; interpreted holons bundle it in `dist/`.
2. **Execution:** The `serve` command must auto-register `HolonMeta.Describe`, answering with the embedded snapshot without requiring local source `.proto` files.

[^1]: `github.com/bufbuild/protocompile`: Pure Go compiler embedded in `op`. Reads proto manifests without `protoc`.
